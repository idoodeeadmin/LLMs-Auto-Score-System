3.3.1 คำอธิบายเอนทิตีภายนอก
ตารางที่ 3.1 คำอธิบายเอนทิตีภายนอก
ชื่อเอนทิตี คำอธิบาย ข้อมูลนำเข้า ข้อมูลส่งออก
ผู้ใช้ทั่วไป ผู้ใช้งานที่ยังไม่ได้เข้าสู่ระบบ สามารถสมัครสมาชิก เข้าสู่ระบบ และดำเนินการเกี่ยวกับการยืนยันตัวตนได้

  • ข้อมูลการสมัครสมาชิก

  • ข้อมูลอีเมลสำหรับรีเซ็ตรหัสผ่าน
  • ข้อมูลบัญชีผู้ใช้
ผู้เรียน ผู้ใช้งานที่เข้าสู่ระบบในฐานะผู้เรียน สามารถทำข้อสอบ ดูผลคะแนน จัดการโปรไฟล์ และรับการแจ้งเตือนต่างๆ ได้

  • ข้อมูลคำตอบข้อสอบ
  • ข้อมูลรหัสเข้าร่วมห้องเรียน
  • ข้อมูลสถานะการอ่านแจ้งเตือน
  • ข้อมูลโปรไฟล์ที่เพิ่มและแก้ไข
  • รหัสผู้เรียนและรหัสข้อสอบ
  • ข้อมูลการแจ้งเตือน

  • ข้อมูลบัญชีผู้ใช้
  • ข้อมูลประกาศข่าวสารห้องเรียน
  • ข้อมูลผลการประเมินและข้อเสนอแนะ
  • ข้อมูลผลคะแนน
  • ข้อมูลสถานะการส่งข้อสอบ
  • ข้อมูลห้องเรียน
  • ข้อมูลเวลาคงเหลือในการสอบ
ผู้สอน ผู้ใช้งานที่เข้าสู่ระบบในฐานะผู้สอน สามารถสร้างห้องเรียน จัดการข้อสอบ เกณฑ์การประเมิน อนุมัติผลคะแนน และแจ้งข่าวสารได้
  • ข้อมูลข้อสอบที่ลบ
  • ข้อมูลข้อสอบที่เพิ่มและแก้ไข
  • ข้อมูลคะแนนและคอมเมนต์ที่เพิ่มและแก้ไข
  • ข้อมูลประกาศข่าวสารห้องเรียนที่ลบ
  • ข้อมูลประกาศข่าวสารห้องเรียนที่เพิ่มและแก้ไข
  • ข้อมูลสถานะการอนุมัติผลคะแนน
  • ข้อมูลสถานะการอ่านแจ้งเตือน
  • ข้อมูลห้องเรียนที่ลบ
  • ข้อมูลห้องเรียนที่เพิ่มและแก้ไข
  • ข้อมูลเกณฑ์การประเมินที่ลบ
  • ข้อมูลเกณฑ์การประเมินที่เพิ่มและแก้ไข
  • ข้อมูลโปรไฟล์และรูปภาพที่เพิ่มและแก้ไข
  • รหัสห้องเรียนและรหัสข้อสอบ
  • ข้อมูลการแจ้งเตือน
  • ข้อมูลข้อสอบ
  • ข้อมูลคำตอบของนักเรียน
  • ข้อมูลบัญชีผู้ใช้
  • ข้อมูลประกาศข่าวสารห้องเรียน
  • ข้อมูลผลการประเมินและข้อเสนอแนะอัจฉริยะ
  • ข้อมูลผลคะแนนของนักเรียน
  • ข้อมูลรายการส่งข้อสอบ
  • ข้อมูลรายงานสรุปคะแนน
  • ข้อมูลสมาชิกในห้องเรียนและสถานะการส่งงาน
  • ข้อมูลห้องเรียน
  • ข้อมูลเกณฑ์การให้คะแนนจาก AI
Firebase Auth บริการภายนอกสำหรับตรวจสอบสิทธิ์และจัดการการยืนยันตัวตนผู้ใช้งานผ่านบัญชี Google
  • Authentication Token
  • ข้อมูลโปรไฟล์ผู้ใช้
  • ข้อมูล Tokens ตรวจสอบสิทธิ์
  • ข้อมูล Tokens บัญชี Google
SMTP เซิร์ฟเวอร์ภายนอกสำหรับจัดการและส่งอีเมลแจ้งเตือน เช่น การยืนยันตัวตน และรีเซ็ตรหัสผ่าน
  • ข้อมูลสถานะการส่งอีเมล
  • ข้อมูลอีเมลและลิงก์ยืนยันตัวตน
  • ข้อมูลอีเมลและลิงก์รีเซ็ตรหัสผ่าน
Cloudinary API บริการภายนอกสำหรับจัดเก็บและจัดการไฟล์รูปภาพบนคลาวด์ เช่น โปรไฟล์ และภาพข้อสอบ
  • ข้อมูล URL และสถานะการจัดเก็บรูปภาพ
  • ข้อมูลไฟล์รูปภาพคำตอบลายมือเขียน
  • ข้อมูลไฟล์รูปภาพโจทย์ข้อสอบ
  • ข้อมูลไฟล์รูปภาพโปรไฟล์
Gemini API บริการ AI ภายนอกสำหรับการตรวจข้อสอบอัตโนมัติ ร่างเกณฑ์การประเมิน และให้ข้อเสนอแนะแก่ผู้เรียน
  • ข้อมูลข้อเสนอแนะการประเมิน
  • ข้อมูลผลการประเมิน
  • ข้อมูลระดับความมั่นใจของคำตอบ
  • ข้อมูลเกณฑ์รูบริคอัตโนมัติ
  • ข้อมูลคำตอบของนักเรียน
  • ข้อมูลบริบทสำหรับการสร้างเกณฑ์
  • ข้อมูลเงื่อนไขและข้อกำหนดการตรวจ
  • ข้อมูลโจทย์ข้อสอบและเกณฑ์การให้คะแนน
3.3.3 คำอธิบายแหล่งเก็บข้อมูลและโครงสร้างข้อมูล
ตารางที่ 3.2 คำอธิบายแหล่งเก็บข้อมูลและโครงสร้างข้อมูล
รหัส ชื่อแฟ้มข้อมูล คำอธิบาย โครงสร้างข้อมูล
D1 แฟ้มข้อมูลผู้ใช้ (users) เก็บรายละเอียดข้อมูลของผู้ใช้งานระบบ รหัส (id)
+ อีเมล (email)
+ รหัสผ่าน (password)
+ ชื่อ (name)
+ บทบาท (role)
+ รหัสนักศึกษา (student_id)
+ ลิงก์รูปโปรไฟล์ (avatar_url)
+ สถานะการยืนยันตัวตน (is_verified)
+ เวอร์ชันโทเคน (token_version)
D2 แฟ้มข้อมูลห้องเรียน (rooms) เก็บรายละเอียดข้อมูลของห้องเรียนที่สร้างโดยผู้สอน รหัส (id)
+ ชื่อห้องเรียน (name)
+ ตอนเรียน (section)
+ รหัสเข้าห้อง (class_code)
+ รหัสผู้สอน (owner_id)
D3 แฟ้มข้อมูลสมาชิกห้องเรียน (enrollments) เก็บรายละเอียดข้อมูลการเข้าร่วมห้องเรียนของผู้เรียน รหัส (id)
+ รหัสผู้ใช้ (user_id)
+ รหัสห้องเรียน (room_id)
+ วันที่เข้าร่วม (joined_at)
D4 แฟ้มข้อมูลข้อสอบ (exams) เก็บรายละเอียดข้อมูลหลักของข้อสอบและการตั้งค่า รหัส (id)
+ รหัสห้องเรียน (room_id)
+ ชื่อข้อสอบ (title)
+ คำอธิบาย (description)
+ คะแนนรวม (total_score)
+ วันที่เริ่มสอบ (start_date)
+ วันที่สิ้นสุดการสอบ (end_date)
+ สุ่มโจทย์ (is_randomized)
+ วันที่สร้าง (created_at)
D5 แฟ้มข้อมูลโจทย์และเกณฑ์ (questions) เก็บรายละเอียดข้อมูลโจทย์แต่ละข้อและเกณฑ์รูบริค (Rubrics) รหัส (id)
+ รหัสข้อสอบ (exam_id)
+ ข้อความโจทย์ (text)
+ คะแนน (score)
+ เฉลย (answer_key)
+ เกณฑ์การประเมิน (rubrics)
+ ลำดับโจทย์ (order_index)
+ รูปภาพ (image_path)
+ หลายรูปภาพ (image_paths)
D6 แฟ้มข้อมูลการส่งข้อสอบ (submissions) เก็บรายละเอียดข้อมูลสถานะและภาพรวมการส่งงานของผู้เรียน รหัส (id)
+ รหัสข้อสอบ (exam_id)
+ รหัสนักศึกษา (student_id)
+ สถานะการส่ง (status)
+ คะแนนรวม (total_score)
+ วันที่ส่ง (submitted_at)
+ ตรวจโดย AI (graded_by_ai)
D7 แฟ้มข้อมูลการตรวจคำตอบ (submission_answers) เก็บข้อมูลคำตอบของผู้เรียน ผลประเมินจาก AI และคะแนนสุทธิจากผู้สอน (ยุบรวม D7, D8, D9) รหัส (id)
+ รหัสการส่ง (submission_id)
+ รหัสโจทย์ (question_id)
+ ข้อความคำตอบ (answer_text)
+ คะแนนจาก AI (ai_score)
+ ข้อเสนอแนะจาก AI (ai_feedback)
+ ระดับความมั่นใจ AI (ai_confidence)
+ คะแนนจากผู้สอน (teacher_score)
+ ความเห็นผู้สอน (teacher_comment)
+ รูปภาพ (image_path)
+ หลายรูปภาพ (image_paths)
+ ตัวชี้วัดคุณภาพ (quality_metrics)
D8 แฟ้มข้อมูลประกาศข่าวสาร (announcements) เก็บรายละเอียดข้อมูลการประกาศแจ้งเตือนภายในห้องเรียน รหัส (id)
+ รหัสห้องเรียน (room_id)
+ รหัสผู้สอน (teacher_id)
+ หัวข้อ (title)
+ เนื้อหา (content)
+ วันที่สร้าง (created_at)
D9 แฟ้มข้อมูลสถานะการอ่านประกาศ (announcement_reads) เก็บสถานะการรับรู้ข่าวสารของผู้เรียนแต่ละคน (Read Receipt) รหัส (id)
+ รหัสประกาศ (announcement_id)
+ รหัสนักศึกษา (student_id)
+ วันที่อ่าน (read_at)
D10 แฟ้มข้อมูลรายละเอียดการแจ้งเตือน (notifications) เก็บรายละเอียดการแจ้งเตือนที่ระบบสร้างขึ้น รหัส (id)
+ รหัสผู้ใช้ (user_id)
+ ประเภท (type)
+ ข้อความ (message)
+ ลิงก์ (link)
+ ข้อมูลเพิ่มเติม (data)
+ สถานะการอ่าน (is_read)
+ วันที่สร้าง (created_at)
3.3.3 คำอธิบายและโครงสร้างของเส้นข้อมูล (Data Flow Description and Data Structure)
ตารางที่ 3.3 Data Flow Description and Data Structure
ชื่อเส้นข้อมูล คำอธิบาย แหล่งต้นทาง แหล่งปลายทาง โครงสร้างข้อมูล
ข้อมูลการสมัครสมาชิก ข้อมูลรายละเอียดส่วนบุคคลและบัญชีที่ผู้ใช้งานกรอกเพื่อขอลงทะเบียนเข้าสู่ระบบ ผู้เรียน Process 1.1 สมัครสมาชิก อีเมล (email)
+ รหัสผ่าน (password)
+ ชื่อ (name)
ข้อมูลผู้ใช้งาน ข้อมูลสิทธิ์ผู้เรียนที่ดึงจากฐานข้อมูลเพื่อตรวจสอบสถานะก่อนอนุญาตให้เข้าทำข้อสอบ Process 1.1 สมัครสมาชิก ผู้เรียน รหัส (id)
+ อีเมล (email)
+ รหัสผ่าน (password)
+ ชื่อ (name)
+ บทบาท (role)
+ รหัสนักศึกษา (student_id)
+ ลิงก์รูปโปรไฟล์ (avatar_url)
+ สถานะการยืนยันตัวตน (is_verified)
+ เวอร์ชันโทเคน (token_version)
D1 แฟ้มข้อมูลผู้ใช้งาน Process 2.1 จัดการข้อมูลห้องเรียน
D1 แฟ้มข้อมูลผู้ใช้งาน Process 5.1 จัดการการเริ่มทำข้อสอบและจับเวลา
ข้อมูลสถานะการส่งอีเมล ข้อมูลสถานะตอบกลับจากเซิร์ฟเวอร์เมล์เพื่อยืนยันว่าส่งอีเมลสำเร็จหรือล้มเหลว Process 1.1 สมัครสมาชิก Process 1.3 ยืนยันอีเมล รหัส (id)
+ อีเมล (email)
+ โทเคน (token)
+ วันที่สร้าง (created_at)
SMTP Server Process 1.3 ยืนยันอีเมล
ข้อมูลการเข้าสู่ระบบ ข้อมูลอีเมลและรหัสผ่านที่ผู้ใช้งานกรอกเพื่อขอตรวจสอบสิทธิ์เข้าสู่ระบบ ผู้เรียน Process 1.2 เข้าสู่ระบบ อีเมล (email)
+ รหัสผ่าน (password)
ข้อมูลคำขอเข้าสู่ระบบด้วยบัญชี Google ข้อมูลที่ใช้ในระบบ Firebase Auth Process 1.2 เข้าสู่ระบบ โทเคน (token)
+ อีเมล (email)
+ สถานะการส่ง (status)
Authentication Token ข้อมูลสถานะการตรวจสอบสิทธิ์และโทเคนยืนยันตัวตนที่ส่งกลับระหว่างระบบกับ Firebase Process 1.2 เข้าสู่ระบบ Firebase Auth โทเคน (token)
+ ชื่อ (name)
+ อีเมล (email)
+ ลิงก์รูปโปรไฟล์ (avatar_url)
+ สถานะการส่ง (status)
ข้อมูลอีเมลและลิงก์ยืนยันตัวตน รหัสยืนยันตัวตนที่ผู้ใช้กดผ่านลิงก์ในอีเมลเพื่อยืนยันความเป็นเจ้าของบัญชี Process 1.3 ยืนยันอีเมล SMTP Server อีเมล (email)
+ หัวข้อ (title)
+ เนื้อหา (content)

ผู้เรียน Process 1.3 ยืนยันอีเมล
