3.8 คำอธิบายการประมวลผล (Process Description)

3.8.1 Process สมัครสมาชิก

Number 1.1
Name สมัครสมาชิก
Description ลงทะเบียนเข้าใช้งานระบบ
Input data flow - ข้อมูลการสมัครสมาชิก
Output data flow - ข้อมูลบัญชีผู้ใช้
- ข้อมูลบัญชีผู้ใช้และโปรไฟล์
Process description เริ่มต้น
1. รับข้อมูลการลงทะเบียนของผู้ใช้ใหม่
2. ตรวจสอบรายละเอียดของข้อมูล เพื่อบันทึกข้อมูล
    2.1 ถ้า (ข้อมูลถูกต้องครบถ้วน)
        2.1.1 ตรวจสอบอีเมล
            - ถ้า (อีเมลซ้ำกับในฐานข้อมูล) แสดงข้อความแจ้งเตือนว่ามีอีเมลนี้แล้ว กลับไป 1.
            - ถ้า (อีเมลไม่ซ้ำในฐานข้อมูล) บันทึกข้อมูลลงฐานข้อมูล
    2.2 ถ้า (ข้อมูลไม่ถูกต้องและไม่ครบถ้วน)
        แสดงข้อความว่าข้อมูลไม่ถูกต้องหรือไม่ครบถ้วน กลับไป 1.
จบการทำงาน

3.8.2 Process เข้าสู่ระบบ

Number 1.2
Name เข้าสู่ระบบ
Description ตรวจสอบสิทธิ์การเข้าใช้งานระบบ
Input data flow - ข้อมูลการเข้าสู่ระบบ
- ข้อมูลบัญชีผู้ใช้เพื่อตรวจสอบ
- ข้อมูลโปรไฟล์ผู้ใช้
Output data flow - ข้อมูลการยืนยันตัวตน
- ข้อมูลบัญชีผู้ใช้
- ข้อมูลบัญชีผู้ใช้และโปรไฟล์
Process description เริ่มต้น
1. รับข้อมูลการเข้าสู่ระบบ (อีเมลและรหัสผ่าน)
2. ตรวจสอบข้อมูลในฐานข้อมูล
    2.1 ถ้าข้อมูลถูกต้องและยืนยันอีเมลแล้ว
        อนุญาตให้เข้าสู่ระบบและสร้างเซสชันการใช้งาน
    2.2 ถ้าข้อมูลไม่ถูกต้อง
        แสดงข้อความเตือนอีเมลหรือรหัสผ่านผิด
จบการทำงาน

3.8.3 Process ยืนยันตัวตนและรีเซ็ตรหัสผ่าน

Number 1.3
Name ยืนยันตัวตนและรีเซ็ตรหัสผ่าน
Description ตรวจสอบโทเคนยืนยันอีเมลหรือรีเซ็ตรหัสผ่านใหม่
Input data flow - ข้อมูลการยืนยันตัวตนทางอีเมล
- ข้อมูลยืนยันตัวตน
- ข้อมูลสถานะการส่งอีเมล
- ข้อมูลอีเมลสำหรับรีเซ็ตรหัสผ่าน
Output data flow - ข้อมูลยืนยันตัวตน
- ข้อมูลอีเมลสำหรับรีเซ็ตรหัสผ่าน
- ข้อมูลอีเมลและลิงก์ยืนยันตัวตน
- ข้อมูลอีเมลและลิงก์รีเซ็ตรหัสผ่าน
Process description เริ่มต้น
1. รับโทเคนหรือข้อมูลการขอรีเซ็ตรหัสผ่าน
2. ตรวจสอบโทเคนในฐานข้อมูล
    2.1 ถ้าโทเคนถูกต้องและยังไม่หมดอายุ
        อัปเดตสถานะการยืนยันตัวตน หรือเปลี่ยนรหัสผ่านใหม่
    2.2 ถ้าโทเคนหมดอายุหรือไม่ถูกต้อง
        แสดงข้อผิดพลาดให้ผู้ใช้ทราบ
จบการทำงาน

3.8.4 Process แก้ไขโปรไฟล์ผู้ใช้

Number 1.4
Name แก้ไขโปรไฟล์ผู้ใช้
Description ปรับปรุงข้อมูลส่วนตัวของผู้ใช้งาน
Input data flow - ข้อมูลผลการจัดเก็บรูปภาพ
- ข้อมูลโปรไฟล์ที่เพิ่มและแก้ไข
Output data flow - ข้อมูลบัญชีผู้ใช้
- ข้อมูลบัญชีผู้ใช้และโปรไฟล์
- ข้อมูลรูปภาพโปรไฟล์
Process description เริ่มต้น
1. รับข้อมูลโปรไฟล์ใหม่จากผู้ใช้
2. ตรวจสอบความถูกต้องของข้อมูล
    2.1 ถ้าข้อมูลถูกต้อง อัปเดตแฟ้มข้อมูลผู้ใช้
    2.2 ถ้าข้อมูลไม่ถูกต้อง แจ้งเตือนข้อผิดพลาด
จบการทำงาน

3.8.5 Process จัดการข้อมูลห้องเรียน

Number 2.1
Name จัดการข้อมูลห้องเรียน
Description สร้างหรือแก้ไขข้อมูลรายละเอียดของห้องเรียน
Input data flow - ข้อมูลบัญชีผู้ใช้เพื่อตรวจสอบ
- ข้อมูลรายละเอียดห้องเรียน
- ข้อมูลห้องเรียนที่ลบ
- ข้อมูลห้องเรียนที่เพิ่มและแก้ไข
Output data flow - ข้อมูลรายละเอียดห้องเรียน
- ข้อมูลห้องเรียน
Process description เริ่มต้น
1. รับคำสั่งและข้อมูลห้องเรียนจากผู้สอน
2. ตรวจสอบสิทธิ์และข้อมูลที่ส่งมา
    2.1 ถ้าเป็นการสร้างใหม่ ให้สร้างรหัสเข้าห้องเรียนและบันทึก
    2.2 ถ้าเป็นการแก้ไข ให้อัปเดตข้อมูลที่มีอยู่
จบการทำงาน

3.8.6 Process เข้าร่วมและดูสมาชิกห้องเรียน

Name เข้าร่วมและดูสมาชิกห้องเรียน

        เพิ่มผู้เรียนเข้าสู่แฟ้มข้อมูลสมาชิกห้องเรียน

- ข้อมูลผลคะแนนที่อนุมัติ
- ข้อมูลผลคะแนนที่แก้ไข
- ข้อมูลสถานะอนุมัติ
Process description เริ่มต้น
1. ดึงผลคะแนนรวมและรายข้อมาแสดงให้ผู้สอนดู
2. รับการตรวจสอบและแก้ไขจากผู้สอน
3. รับคำสั่งยืนยันการประกาศผลคะแนน
4. อัปเดตสถานะให้ผู้เรียนสามารถดูผลสอบได้

3.8.17 Process จัดการแจ้งเตือนและส่งออกสถิติ

Number 7.0
Name จัดการแจ้งเตือนและส่งออกสถิติ
Description ส่งการแจ้งเตือนสถานะและทำรายงานสถิติ
Input data flow - ข้อมูลคะแนนทางการ
- ข้อมูลประวัติการแจ้งเตือน
- ข้อมูลผลการประเมินจาก AI
- ข้อมูลผลคะแนนรวม
- ข้อมูลผลคะแนนรายข้อ
- ข้อมูลรายละเอียดข้อสอบ
- ข้อมูลสถานะการอ่านประกาศ
- ข้อมูลสถานะการอ่านแจ้งเตือน
- รหัสผู้เรียนและรหัสข้อสอบ
- รหัสห้องเรียนและรหัสข้อสอบ
Output data flow - ข้อมูลการแจ้งเตือน
- ข้อมูลการแจ้งเตือนใหม่
- ข้อมูลผลการประเมินและข้อเสนอแนะ
- ข้อมูลผลคะแนน
- ข้อมูลผลคะแนนของนักเรียน
- ข้อมูลรายงานสรุปคะแนน
- ข้อมูลสถานะการอ่านล่าสุด
- ข้อมูลสถานะการอ่านแจ้งเตือน
Process description เริ่มต้น
1. ตรวจสอบเหตุการณ์ใหม่ในระบบ (เช่น คะแนนออก, มีประกาศ)
2. สร้างข้อความแจ้งเตือนและส่งให้ผู้เรียน/ผู้สอนที่เกี่ยวข้อง
3. หากมีคำสั่งส่งออกสถิติ ให้ประมวลผลคะแนนแล้วออกรายงาน
