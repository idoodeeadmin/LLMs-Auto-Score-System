3.8 Process Specification (คำอธิบายการประมวลผล)

3.8.1 Process สมัครสมาชิก

[ใส่รูปภาพ DFD ของ Process 1.1 ที่นี่]
Number 1.1
Name สมัครสมาชิก
Description กระบวนการสมัครสมาชิกของระบบ
Input data flow
  • ข้อมูลการสมัครสมาชิก
Output data flow
  • ข้อมูลผู้ใช้งาน
  • ข้อมูลสถานะการส่งอีเมล
Process description
เริ่มต้น
1. รับข้อมูลการสมัครสมาชิกจากผู้ใช้
2. ตรวจสอบรายละเอียดของข้อมูล
2.1 ถ้า(ข้อมูลรูปแบบถูกต้องและอีเมลไม่ซ้ำในระบบ)
2.1.1 บันทึกข้อมูลบัญชีผู้ใช้ใหม่ลงในแฟ้มข้อมูลผู้ใช้งาน (D1)
2.1.2 สร้างโทเคนสำหรับการยืนยันอีเมล
2.1.3 ส่งข้อมูลสถานะการส่งอีเมลไปยัง Process 1.3
2.2 ถ้า(ข้อมูลไม่ถูกต้องหรืออีเมลซ้ำ)
2.2.1 แสดงข้อความแจ้งเตือนให้ผู้ใช้แก้ไข
จบการทำงาน

3.8.2 Process เข้าสู่ระบบ

[ใส่รูปภาพ DFD ของ Process 1.2 ที่นี่]
Number 1.2
Name เข้าสู่ระบบ
Description กระบวนการเข้าสู่ระบบของระบบ
Input data flow
  • ข้อมูลการเข้าสู่ระบบ
  • ข้อมูลคำขอเข้าสู่ระบบด้วยบัญชี Google
Output data flow
  • Authentication Token
Process description
เริ่มต้น
1. รับข้อมูลการเข้าสู่ระบบ (อีเมลและรหัสผ่าน) หรือข้อมูลคำขอเข้าสู่ระบบด้วยบัญชี Google
2. ตรวจสอบรายละเอียดของข้อมูล
2.1 กรณีเข้าสู่ระบบด้วยอีเมล
2.1.1 ตรวจสอบข้อมูลในฐานข้อมูล (D1)
2.1.2 ถ้า(อีเมลและรหัสผ่านถูกต้อง และยืนยันตัวตนแล้ว)
- สร้าง Authentication Token แจ้งเข้าสู่ระบบสำเร็จ
2.1.3 ถ้า(ไม่ถูกต้อง)
- แสดงข้อความเตือนข้อมูลไม่ถูกต้อง
2.2 กรณีเข้าสู่ระบบด้วย Google
2.2.1 ส่งโทเคนไปตรวจสอบกับ Firebase Auth
2.2.2 ถ้า(ยืนยันสำเร็จ) สร้าง Authentication Token ให้ผู้ใช้
จบการทำงาน

3.8.3 Process ยืนยันอีเมล

[ใส่รูปภาพ DFD ของ Process 1.3 ที่นี่]
Number 1.3
Name ยืนยันอีเมล
Description กระบวนการยืนยันอีเมลของระบบ
Input data flow
  • ข้อมูลสถานะการส่งอีเมล
  • ข้อมูลอีเมลและลิงก์ยืนยันตัวตน
Output data flow
  • ข้อมูลอีเมลและลิงก์ยืนยันตัวตน
Process description
เริ่มต้น
1. รับข้อมูลสถานะการส่งอีเมล หรือ ข้อมูลลิงก์ยืนยันตัวตน
2. ตรวจสอบประเภทคำสั่ง
2.1 กรณีส่งอีเมล

2.1.2 รอรับรหัสสถานะจากเซิร์ฟเวอร์ และแจ้งผลให้ผู้ใช้ทราบ
2.2 กรณียืนยันตัวตนผ่านลิงก์
2.2.1 ตรวจสอบรหัสโทเคนในฐานข้อมูล
2.2.2 ถ้า(ถูกต้อง) อัปเดตสถานะการยืนยันตัวตนใน D1

3.8.2 Process จัดการการเข้าร่วมห้องเรียน

Number 2.2
Name จัดการการเข้าร่วมห้องเรียน

2.1.3 แจ้งผลการเข้าร่วมสำเร็จ
2.2 ถ้า(รหัสไม่ถูกต้อง)
