คำอธิบายกระบวนการ (Process Description)

Level 1: System Processes
กระบวนการหลักระดับบนสุดของระบบ LLMs Auto Score System
1.0
จัดการบัญชีผู้ใช้งาน
กระบวนการควบคุมสิทธิ์การเข้าถึง สมัครสมาชิก เข้าสู่ระบบ ยืนยันอีเมล และรีเซ็ตรหัสผ่าน โดยเชื่อมต่อกับ Firebase Auth และบริการ SMTP
2.0
จัดการห้องเรียน
กระบวนการสร้างและแก้ไขห้องเรียนสำหรับผู้สอน รวมถึงการเข้าร่วมห้องเรียนและจัดการสถานะสมาชิกสำหรับผู้เรียน
3.0
จัดการข้อสอบและเกณฑ์ประเมิน
กระบวนการสร้างชุดข้อสอบ เพิ่มโจทย์ อัปโหลดรูปภาพโจทย์ (Cloudinary) และการใช้ AI (Gemini) สร้างเกณฑ์รูบริคแบบอัตโนมัติ
4.0
ดำเนินการสอบ
กระบวนการสำหรับผู้เรียนในการโหลดข้อสอบ บันทึกคำตอบชั่วคราว อัปโหลดรูปภาพคำตอบ และส่งข้อสอบเข้าสู่ระบบ
5.0
ประเมินผลคำตอบด้วย AI
กระบวนการเชื่อมต่อ Gemini API เพื่อวิเคราะห์คำตอบของผู้เรียนเทียบกับเกณฑ์รูบริค พร้อมประมวลผลผลการประเมินและคำแนะนำ
6.0
จัดการผลคะแนนและรายงาน
กระบวนการสำหรับผู้สอนในการตรวจสอบ อนุมัติผลคะแนนที่ AI ตรวจสอบแล้ว รวมถึงการส่งผลคะแนนให้ผู้เรียนและออกรายงานสถิติ
7.0
จัดการแจ้งเตือนและการสื่อสาร
กระบวนการแจ้งเตือนแบบเรียลไทม์ เช่น แจ้งเตือนข้อสอบใหม่, แจ้งเตือนหมดเวลา, และแจ้งเตือนเมื่อประกาศผลคะแนน
Level 2: Sub-processes
กระบวนการย่อยที่แตกรายละเอียด (Breakdown) ของกระบวนการหลักใน Level 1 ตามพฤติกรรมการใช้งาน (Action-Oriented)
1.1
ลงทะเบียนผู้ใช้งานใหม่
รับข้อมูลการสมัครสมาชิกจากผู้ใช้ทั่วไป และบันทึกข้อมูลโปรไฟล์ใหม่ลงระบบ
1.2
ตรวจสอบสิทธิ์การเข้าสู่ระบบ
ตรวจสอบข้อมูลประจำตัว และจ่ายข้อมูลบัญชีผู้ใช้เมื่อเข้าสู่ระบบสำเร็จ
1.3
ตรวจสอบสิทธิ์ด้วยบัญชีภายนอก
เชื่อมต่อ Firebase Auth เพื่อตรวจสอบ Token และดึงข้อมูลโปรไฟล์ผู้ใช้มาบันทึกลงระบบ
1.4
ประมวลผลการยืนยันอีเมล
เชื่อมต่อระบบ SMTP เพื่อส่งอีเมลและลิงก์ยืนยันตัวตนไปยังผู้ใช้งาน
1.5
ประมวลผลการรีเซ็ตรหัสผ่าน
เชื่อมต่อระบบ SMTP เพื่อส่งลิงก์รีเซ็ตรหัสผ่าน และอัปเดตรหัสผ่านใหม่ลงในแฟ้มข้อมูลโปรไฟล์
2.1
บันทึกข้อมูลห้องเรียนใหม่
รับข้อมูลจากผู้สอน และบันทึกข้อมูลห้องเรียนใหม่ลงในแฟ้มข้อมูลห้องเรียน
2.2
ปรับปรุงข้อมูลห้องเรียน
จัดการอัปเดต หรือลบข้อมูลห้องเรียนตามคำสั่งของผู้สอน
2.3
ประมวลผลการเข้าร่วมห้องเรียน
ตรวจสอบรหัสห้องเรียน และเพิ่มข้อมูลสมาชิกใหม่ลงในแฟ้มข้อมูลสมาชิก
2.4
ปรับปรุงข้อมูลสมาชิกห้องเรียน
ดึงข้อมูลและแสดงรายชื่อสมาชิก รวมถึงสถานะการส่งงานให้ผู้สอนทราบ
3.1
บันทึกข้อมูลรายละเอียดข้อสอบ
สร้างข้อมูลข้อสอบใหม่ กำหนดระยะเวลา และผูกข้อสอบเข้ากับห้องเรียน
3.2
ปรับปรุงข้อมูลโจทย์และเกณฑ์
เพิ่ม ลบ และอัปเดตข้อมูลโจทย์และเกณฑ์การประเมินลงในแฟ้มข้อมูล
3.3
บันทึกไฟล์รูปภาพประกอบข้อสอบ
ส่งรูปภาพโจทย์ข้อสอบไปยัง Cloudinary API และบันทึก URL กลับมาที่ระบบ
3.4
สร้างเกณฑ์การประเมินด้วย AI
ส่งข้อมูลบริบทโจทย์ไปให้ Gemini API เพื่อรับเกณฑ์รูบริคอัตโนมัติกลับมาใช้งาน
4.1
ดึงข้อมูลข้อสอบสำหรับผู้เรียน
ดึงรายละเอียดข้อสอบ พร้อมคำนวณและแสดงผลเวลาคงเหลือในการสอบให้ผู้เรียน
4.2
บันทึกคำตอบระหว่างสอบ
รับคำตอบที่ผู้เรียนพิมพ์และบันทึกเป็นฉบับร่างชั่วคราวลงแฟ้มข้อมูลคำตอบ
4.3
บันทึกไฟล์รูปภาพคำตอบ
ส่งรูปภาพคำตอบลายมือเขียนหรือวิธีทำไปยัง Cloudinary API เพื่อนำ URL มาประกอบการส่ง
4.4
บันทึกสถานะการส่งข้อสอบ
ยืนยันการส่งข้อสอบ อัปเดตสถานะการส่ง และเปลี่ยนสถานะเพื่อรอการประเมิน
5.1
รวบรวมข้อมูลสำหรับการตรวจ
ดึงข้อมูลโจทย์ เกณฑ์รูบริค และคำตอบของนักเรียนจากฐานข้อมูลมาเตรียมพร้อม
5.2
ส่งข้อมูลเข้าสู่ระบบ AI
นำข้อมูลทั้งหมดที่เตรียมไว้ ส่งไปยัง Gemini API พร้อมเงื่อนไขการตรวจ
5.3
ประมวลผลผลการประเมิน
รับผลลัพธ์การประเมินดิบ ระดับความมั่นใจ และข้อเสนอแนะกลับมาจาก AI
5.4
บันทึกผลการประเมิน
จัดรูปแบบผลลัพธ์ บันทึกคะแนนลงระบบ และอัปเดตสถานะว่าตรวจเสร็จสิ้นแล้ว
