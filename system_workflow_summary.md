# 🚀 LLMs Auto Score System - Comprehensive Master Document\nเอกสารนี้คือสรุปแบบละเอียดของระบบ (Single Source of Truth) เพื่อลดการใช้ Tokens ในการค้นหาไฟล์ HTML โดยรวมข้อมูลทั้งหมดตั้งแต่ DFD Level 0 ถึง Level 2, Data Stores และ Data Flows ไว้ในไฟล์เดียว\n\n## 🌍 1. DFD Level 0 (Context Diagram)\nภาพรวมสูงสุดของระบบและการสื่อสารกับ External Entities ทั้งหมด\n\n\n\n## ⚙️ 2. DFD Level 1 (Core Processes)\nการแบ่งระบบออกเป็น 7 Process หลัก และการสื่อสารกับ Data Stores และ Entities\n\n\n\n## 🔬 3. DFD Level 2 (Sub-processes)\nการแตกรายละเอียดการทำงานย่อย (Sub-processes) ภายในแต่ละ Process หลัก\n\n\n\n## 🗄️ 4. Data Stores (แฟ้มข้อมูลทั้งหมด)
\n\n| ID | Name | Description | Structure |
\n\n|----|------|-------------|-----------|
\n\n| **D1** | แฟ้มข้อมูลผู้ใช้งาน (users) | เก็บรายละเอียดข้อมูลของผู้ใช้งานระบบ | รหัส (id) + อีเมล (email) + รหัสผ่าน (password) + ชื่อ (name) + บทบาท (role) + รหัสนักศึกษา (student_id) + ลิงก์รูปโปรไฟล์ (avatar_url) + สถานะการยืนยันตัวตน (is_verified) + เวอร์ชันโทเคน (token_version) |
\n\n| **D2** | แฟ้มข้อมูลห้องเรียน (rooms) | เก็บรายละเอียดข้อมูลของห้องเรียนที่สร้างโดยผู้สอน | รหัส (id) + ชื่อห้องเรียน (name) + ตอนเรียน (section) + รหัสเข้าห้อง (class_code) + รหัสผู้สอน (owner_id) |
\n\n| **D3** | แฟ้มข้อมูลสมาชิกห้องเรียน (enrollments) | เก็บรายละเอียดข้อมูลการเข้าร่วมห้องเรียนของผู้เรียน | รหัส (id) + รหัสผู้ใช้ (user_id) + รหัสห้องเรียน (room_id) + วันที่เข้าร่วม (joined_at) |
\n\n| **D4** | แฟ้มข้อมูลข้อสอบ (exams) | เก็บรายละเอียดข้อมูลหลักของข้อสอบและการตั้งค่า | รหัส (id) + รหัสห้องเรียน (room_id) + ชื่อข้อสอบ (title) + คำอธิบาย (description) + คะแนนรวม (total_score) + วันที่เริ่มสอบ (start_date) + วันที่สิ้นสุดการสอบ (end_date) + สุ่มโจทย์ (is_randomized) + วันที่สร้าง (created_at) |
\n\n| **D5** | แฟ้มข้อมูลโจทย์และเกณฑ์ (questions) | เก็บรายละเอียดข้อมูลโจทย์แต่ละข้อและเกณฑ์รูบริค (Rubrics) | รหัส (id) + รหัสข้อสอบ (exam_id) + ข้อความโจทย์ (text) + คะแนน (score) + เฉลย (answer_key) + {เกณฑ์การประเมิน} (rubrics) + ลำดับโจทย์ (order_index) + รูปภาพ (image_path) + {หลายรูปภาพ} (image_paths) |
\n\n| **D6** | แฟ้มข้อมูลการส่งข้อสอบ (submissions) | เก็บรายละเอียดข้อมูลสถานะและภาพรวมการส่งงานของผู้เรียน | รหัส (id) + รหัสข้อสอบ (exam_id) + รหัสนักศึกษา (student_id) + สถานะการส่ง (status) + คะแนนรวม (total_score) + วันที่ส่ง (submitted_at) + ตรวจโดย AI (graded_by_ai) |
\n\n| **D7** | แฟ้มข้อมูลคำตอบ (submission_answers) | เก็บข้อมูลคำตอบของผู้เรียน ผลประเมินจาก AI และคะแนนสุทธิจากผู้สอน | รหัส (id) + รหัสการส่ง (submission_id) + รหัสโจทย์ (question_id) + ข้อความคำตอบ (answer_text) + คะแนนจาก AI (ai_score) + {ข้อเสนอแนะจาก AI} (ai_feedback) + ระดับความมั่นใจ AI (ai_confidence) + คะแนนจากผู้สอน (teacher_score) + ความเห็นผู้สอน (teacher_comment) + รูปภาพ (image_path) + {หลายรูปภาพ} (image_paths) + ตัวชี้วัดคุณภาพ (quality_metrics) |
\n\n| **D8** | แฟ้มข้อมูลรีเซ็ตรหัสผ่าน (password_resets) | เก็บข้อมูล Token สำหรับการรีเซ็ตรหัสผ่าน | รหัส (id) + รหัสผู้ใช้ (user_id) + โทเคน (token) + วันเวลาที่ใช้งาน (used_at) + วันหมดอายุ (expires_at) + วันที่สร้าง (created_at) |
\n\n| **D13** | แฟ้มข้อมูลยืนยันอีเมล (email_verifications) | เก็บข้อมูล Token สำหรับการยืนยันอีเมล | รหัส (id) + รหัสผู้ใช้ (user_id) + โทเคน (token) + วันเวลาที่ยืนยัน (verified_at) + วันหมดอายุ (expires_at) + วันที่สร้าง (created_at) |
\n\n| **D14** | แฟ้มข้อมูลประกาศข่าวสาร (announcements) | เก็บรายละเอียดข้อมูลการประกาศแจ้งเตือนภายในห้องเรียน | รหัส (id) + รหัสห้องเรียน (room_id) + รหัสผู้สอน (teacher_id) + หัวข้อ (title) + เนื้อหา (content) + วันที่สร้าง (created_at) |
\n\n| **D15** | แฟ้มข้อมูลการอ่านประกาศ (announcement_reads) | เก็บสถานะการรับรู้ข่าวสารของผู้เรียนแต่ละคน (Read Receipt) | รหัส (id) + รหัสประกาศ (announcement_id) + รหัสนักศึกษา (student_id) + วันที่อ่าน (read_at) |
\n\n| **D12** | แฟ้มข้อมูลแจ้งเตือน (notifications) | เก็บรายละเอียดการแจ้งเตือนที่ระบบสร้างขึ้น | รหัส (id) + รหัสผู้ใช้ (user_id) + ประเภท (type) + ข้อความ (message) + ลิงก์ (link) + ข้อมูลเพิ่มเติม (data) + สถานะการอ่าน (is_read) + วันที่สร้าง (created_at) |

\n\n## 👤 5. External Entities (หน่วยข้อมูลภายนอก)\n| Entity | Description |\n|--------|-------------|\n| **ผู้ใช้ทั่วไป** | ผู้ใช้งานที่ยังไม่ได้เข้าสู่ระบบ สามารถสมัครสมาชิก เข้าสู่ระบบ และดำเนินการเกี่ยวกับการยืนยันตัวตนได้ |\n| **ผู้เรียน** | ผู้ใช้งานที่เข้าสู่ระบบในฐานะผู้เรียน สามารถทำข้อสอบ ดูผลคะแนน จัดการโปรไฟล์ และรับการแจ้งเตือนต่างๆ ได้ |\n| **ผู้สอน** | ผู้ใช้งานที่เข้าสู่ระบบในฐานะผู้สอน สามารถสร้างห้องเรียน จัดการข้อสอบ เกณฑ์การประเมิน อนุมัติผลคะแนน และแจ้งข่าวสารได้ |\n| **Firebase Auth** | บริการภายนอกสำหรับตรวจสอบสิทธิ์และจัดการการยืนยันตัวตนผู้ใช้งานผ่านบัญชี Google |\n| **SMTP** | เซิร์ฟเวอร์ภายนอกสำหรับจัดการและส่งอีเมลแจ้งเตือน เช่น การยืนยันตัวตน และรีเซ็ตรหัสผ่าน |\n| **Cloudinary API** | บริการภายนอกสำหรับจัดเก็บและจัดการไฟล์รูปภาพบนคลาวด์ เช่น โปรไฟล์ และภาพข้อสอบ |\n| **Gemini API** | บริการ AI ภายนอกสำหรับการตรวจข้อสอบอัตโนมัติ ร่างเกณฑ์การประเมิน และให้ข้อเสนอแนะแก่ผู้เรียน |\n\n\n## 🔄 6. Data Flow Dictionary (พจนานุกรมการไหลของข้อมูล)
\n\n| Data Flow Name | Description | Structure |
\n\n|----------------|-------------|-----------|
\n\n| **ข้อมูลการสมัครสมาชิก** | ข้อมูลรายละเอียดส่วนบุคคลและบัญชีที่ผู้ใช้งานกรอกเพื่อขอลงทะเบียนเข้าสู่ระบบ | อีเมล + รหัสผ่าน + ชื่อ + บทบาท + รหัสผู้เรียน |
\n\n| **ข้อมูลการเข้าสู่ระบบ** | ข้อมูลอีเมลและรหัสผ่านที่ผู้ใช้งานกรอกเพื่อขอตรวจสอบสิทธิ์เข้าสู่ระบบ | อีเมล + รหัสผ่าน |
\n\n| **ข้อมูลอีเมลสำหรับรีเซ็ตรหัสผ่าน** | ข้อมูลอีเมลของผู้ใช้ที่ต้องการขอเปลี่ยนรหัสผ่านเพื่อสร้าง Token ใหม่ | อีเมล + ชื่อ + รหัสผู้เรียน |
\n\n| **ข้อมูลการยืนยันตัวตนทางอีเมล** | ข้อมูลการยืนยันตัวตนทางอีเมลสำหรับใช้ในกระบวนการตรวจสอบสิทธิ์ ยืนยันตัวตน หรือเป็นช่องทางติดต่อผู้ใช้งาน | รหัสผู้ใช้ + อีเมล + รหัสโทเคน + วันหมดอายุ |
| **ข้อมูลบัญชีผู้ใช้** | ข้อมูลรายละเอียดส่วนบุคคลของผู้ใช้งาน บัญชีผู้ใช้ ที่ส่งกลับไปให้ผู้ใช้งานหลังจากทำรายการหรือสอบถามสำเร็จ | รหัสอ้างอิง + อีเมล + ชื่อ + บทบาท + รหัสผู้เรียน + รูปโปรไฟล์ + สถานะยืนยันอีเมล |
| **ข้อมูลการยืนยันตัวตน** | ชุดข้อมูลสำหรับร้องขอการยืนยันตัวตนกับระบบภายนอก | Google ID Token หรือ (Email + Password) |
\n\n| **รหัสผู้เรียนและรหัสข้อสอบ** | รหัสผู้เรียนและรหัสข้อสอบที่ผู้สอนระบุเข้าระบบเพื่อใช้เป็นเงื่อนไขในการสืบค้น และเรียกดูรายงานสรุปผลคะแนนสอบของนักเรียนเป็นรายบุคคล | รหัสผู้เรียน + รหัสข้อสอบ |
\n\n| **ข้อมูลการส่งข้อสอบ** | ชุดข้อมูลรายละเอียดของการส่งข้อสอบ ที่ใช้สำหรับอ้างอิงในการดึงข้อมูล บันทึก หรือจัดเตรียมเพื่อการแสดงผลในระบบ | รหัสผู้เรียน + รหัสข้อสอบ + {ชุดคำตอบ} + เวลาที่ส่ง |
\n\n| **ข้อมูลคำตอบข้อสอบ** | ชุดข้อมูลคำตอบข้อสอบที่ผู้เรียนส่งเข้าสู่ระบบ เพื่อรวบรวมนำไปเข้าสู่กระบวนการตรวจและคำนวณผลประเมิน | รหัสผู้เรียน + รหัสข้อสอบ + คำตอบ + เวลาที่ส่ง |
\n\n| **ข้อมูลรหัสเข้าร่วมห้องเรียน** | รหัสคลาสเรียน 6 หลักที่ผู้เรียนกรอกเพื่อขอลงทะเบียนเข้าร่วมห้องเรียน | รหัสผู้เรียน + รหัสเข้าร่วมห้องเรียน6หลัก + วันเวลาที่ส่งคำขอ |
\n\n| **ข้อมูลโปรไฟล์ที่เพิ่มและแก้ไข** | ข้อมูลการแก้ไขข้อมูลส่วนตัวและรูปภาพที่ถูกส่งมาจากผู้สอนเพื่ออัปเดตลงฐานข้อมูล | บทบาท + รหัสอ้างอิงประจำตัว |
\n\n| **ข้อมูลห้องเรียน** | ข้อมูลรายละเอียดห้องเรียนที่ดึงมาตรวจสอบสิทธิ์ความเป็นเจ้าของห้องก่อนจัดการข้อสอบ | รหัสอ้างอิง + ชื่อ + กลุ่มเรียน + รหัสห้องเรียน6หลัก + รหัสเจ้าของห้อง + เวลาสร้าง |
\n\n| **ข้อมูลข้อสอบ** | ข้อมูลรายละเอียดของข้อสอบที่ดึงมาเพื่อใช้เป็นส่วนหัว (Header) ของไฟล์รายงาน | รหัสข้อสอบ + ชื่อข้อสอบ + เวลาเริ่มสอบ + เวลาสิ้นสุดการสอบ + คะแนนเต็ม |
\n\n| **ข้อมูลผลคะแนน** | รายละเอียดของผลคะแนน ที่ใช้สำหรับบันทึกลงฐานข้อมูล นำไปแสดงผลให้ผู้ใช้ทราบ หรือใช้สรุปผลสัมฤทธิ์ | รหัสข้อสอบ + รหัสผู้เรียน + คะแนนรวม |
\n\n| **ข้อมูลผลการประเมินและข้อเสนอแนะจาก AI** | รายละเอียดของผลการประเมินและข้อเสนอแนะอัจฉริยะ ที่ใช้สำหรับบันทึกลงฐานข้อมูล นำไปแสดงผลให้ผู้ใช้ทราบ หรือใช้สรุปผลสัมฤทธิ์ | คะแนนจากAI + ข้อเสนอแนะจากAI + ระดับความมั่นใจ + เกณฑ์ที่ตรงกัน |
\n\n| **ข้อมูลประกาศข่าวสารห้องเรียน** | ข้อมูลประกาศข่าวสารห้องเรียนที่ใช้สำหรับบรอดแคสต์ส่งข้อความ หรือแสดงผลอัปเดตความเคลื่อนไหวล่าสุดให้ผู้ใช้งานทราบ | รหัสประกาศ + หัวข้อ + เนื้อหาประกาศ + ผู้สร้าง + เวลาประกาศ |
\n\n| **ข้อมูลการแจ้งเตือนแบบเรียลไทม์** | ข้อมูลข้อความแจ้งเตือนระบบแบบเรียลไทม์ผ่าน Socket.io และสถานะการเปิดอ่านแจ้งเตือนของผู้ใช้งาน | รหัสผู้ใช้ + ประเภทแจ้งเตือน + ข้อความแจ้งเตือน + ข้อมูลเชื่อมโยง |
\n\n| **ข้อมูลสถานะการส่งข้อสอบ** | ข้อมูลระบุสถานะการส่งข้อสอบ เพื่อใช้ในการอัปเดตความคืบหน้า อนุมัติสิทธิ์ หรือปรับปรุงผลลัพธ์ของการทำรายการในระบบ | รหัสอ้างอิง + รหัสข้อสอบ + รหัสผู้เรียน + สถานะ + คะแนนรวม + เวลาที่ส่ง |
\n\n| **ข้อมูลเวลาคงเหลือในการสอบ** | ชุดข้อมูลรายละเอียดของเวลาคงเหลือในการสอบ ที่ใช้สำหรับอ้างอิงในการดึงข้อมูล บันทึก หรือจัดเตรียมเพื่อการแสดงผลในระบบ | รหัสผู้เรียน + รหัสข้อสอบ + เวลาคงเหลือ |

\n\n## 📋 7. กฎเกณฑ์และมาตรฐานการเขียน DFD (DFD Rules & Standards)
\n\nข้อกำหนดต่อไปนี้อ้างอิงจากหลักการเขียน DFD เพื่อใช้ตรวจสอบความถูกต้องของเอกสาร:
\n\n
\n\n1. **Process (กระบวนการ)**
\n\n   - ต้องเขียนชื่อเป็น **Verb + Noun (กริยา + นาม)** เพื่อให้เห็นว่าระบบทำอะไร (เช่น จัดการข้อมูลโปรไฟล์, สมัครสมาชิก)
\n\n   - หาก Process สามารถแตกรายละเอียดได้ ให้แตกย่อยในระดับถัดไป (Level 2)
\n\n2. **Data Flow (เส้นการไหลของข้อมูล)**
\n\n   - ใช้ลูกศรบอกทิศทางว่าข้อมูลใดเข้าหรือออก
\n\n   - ข้อมูลต่างกันต้องใช้ชื่อต่างกัน
\n\n   - **กฎความสมดุล (Balancing Rule):** เส้น Data Flow ในระดับลึกสุด (เช่น Level 2) จะต้องปรากฏเส้นเดียวกันนี้ในระดับที่สูงขึ้นมาจนถึง Context Diagram (Level 0)
\n\n   - จำนวนเส้น Data Flow ต้องมีจำนวนเท่ากันในทุก Level ที่เชื่อมโยงกัน
\n\n3. **Data Store (แหล่งเก็บข้อมูล)**
\n\n   - ใช้เก็บข้อมูลโดยจัดกลุ่มตามความเหมาะสม (ไม่จำเป็นต้องเหมือนโครงสร้าง Table ใน Database จริงเสมอไป)
\n\n   - ควรออกแบบให้มี Data Store จำนวนน้อยแต่แยกกลุ่มข้อมูลได้ชัดเจน
\n\n   - รูปแบบการเขียนอธิบาย (Data Store Description) ประกอบด้วย: ID | ชื่อแฟ้มข้อมูล (Data Store) | โครงสร้างข้อมูล (Structure)
\n\n4. **External Entity (เอนทิตีภายนอก)**
\n\n   - คือผู้เกี่ยวข้องกับระบบ สามารถเป็นได้ทั้ง: คน (User หลายประเภท), อุปกรณ์ (Devices), หรือ ระบบอื่นๆ (External Systems)
\n\n   - **การแบ่งบทบาทผู้ใช้:** หากผู้ใช้ทั่วไปสมัครสมาชิก ให้ใช้ Entity ผู้ใช้ทั่วไป ส่งข้อมูลไปยัง Process สมัครสมาชิก แต่ถ้าแก้ไขข้อมูลส่วนตัวให้ใช้ Entity สมาชิก หรือ ผู้เรียน/ผู้สอน ส่งข้อมูลไปที่ แก้ไขข้อมูลส่วนตัว
\n\n   - ในตารางคำอธิบาย (External Entity Description) จะต้องอ้างอิงจาก Context Diagram โดยมีกฎคือ:
\n\n     - **Input Data Flow (ข้อมูลนำเข้า):** คือเส้นที่เข้ามาหาผู้ใช้นั้น (ลูกศรชี้มาที่ External Entity)
\n\n     - **Output Data Flow (ข้อมูลส่งออก):** คือเส้นที่ออกไปจากผู้ใช้นั้น (ลูกศรชี้ออกจาก External Entity)
\n\n5. **สัญลักษณ์ใน Data Dictionary**
\n\n   - = ประกอบด้วย หรือ เท่ากับ
\n\n   - + และ
\n\n   - { } มีการทำซ้ำของส่วนย่อยข้อมูล
\n\n   - [ | ] เลือกส่วนย่อยของข้อมูลตัวใดตัวหนึ่ง
\n\n   - ( ) ออปชันนอล คือ มีหรือไม่มีก็ได้
\n\n   - * หมายเหตุ
\n\n6. **สัญลักษณ์ซ้ำในแผนภาพ**
\n\n   - หากใน DFD Level เดียวกันมี External Entity หรือ Data Store ที่ซ้ำกัน ให้ทำเครื่องหมายขีดมุมล่างซ้ายเพื่อแสดงการทำซ้ำ

## โพรเซสย่อย 1.0 ถึง 7.0 (Workflow / Trigger-Oriented)

*การออกแบบโครงสร้าง DFD Level 2 ยึดหลัก Workflow มีเส้นเชื่อมระหว่าง Process ย่อย (Internal Links) เพื่อแสดงลำดับการทำงานที่ชัดเจน, ไม่ใช้ Temp Data Store, ชื่อ Process เป็นคำกริยา, Data Flow เป็นคำนาม และมีความสมดุลกับ Level 1 แบบ 100%*

**▶ Process 1.0 จัดการบัญชีผู้ใช้งาน**
- 1.1 สร้างบัญชีผู้ใช้งาน: รับข้อมูลการสมัครสมาชิก สร้างโปรไฟล์ลง D1
- 1.2 ตรวจสอบสิทธิ์เข้าสู่ระบบ: รับข้อมูลเข้าสู่ระบบ ตรวจสอบกับ D1 ส่งผลกลับไป
- 1.3 เชื่อมต่อบัญชี Google: รับข้อมูลจาก Firebase Auth บันทึกลง D1
- 1.4 ตรวจสอบอีเมลยืนยันตัวตน: รับคำขอยืนยัน ส่งอีเมลผ่าน SMTP และบันทึกลง D9
- 1.5 สร้างลิงก์รีเซ็ตรหัสผ่าน: รับคำขอรีเซ็ต ส่งลิงก์ผ่าน SMTP และบันทึกลง D8
- 1.6 บันทึกข้อมูลโปรไฟล์: รับการแก้ไขโปรไฟล์จากผู้ใช้ บันทึกลง D1

**▶ Process 2.0 จัดการห้องเรียน**
- 2.1 สร้างข้อมูลห้องเรียน: ผู้สอนสร้างห้อง บันทึกลง D2 (อิง D1)
- 2.2 ปรับปรุงข้อมูลห้องเรียน: อัปเดต/ลบห้องใน D2
- 2.3 บันทึกสมาชิกห้องเรียน: ผู้เรียนนำรหัสเข้าร่วม บันทึกลง D3
- 2.4 ตรวจสอบสมาชิกห้องเรียน: ผู้สอนดูรายชื่อสมาชิกจาก D3
- 2.5 บันทึกประกาศข่าวสาร: ผู้สอนจัดการประกาศ บันทึกลง D10

**▶ Process 3.0 จัดการข้อสอบและเกณฑ์ประเมิน**
- 3.1 บันทึกโครงสร้างข้อสอบ: ผู้สอนตั้งค่าข้อสอบ บันทึกลง D4
- 3.2 บันทึกโจทย์และเกณฑ์: ผู้สอนเพิ่มโจทย์และรูบริค บันทึกลง D5
- 3.3 จัดเก็บรูปภาพโจทย์: อัปโหลดรูปให้ Cloudinary บันทึกลิงก์ลง D5
- 3.4 รวบรวมเกณฑ์อัตโนมัติ: ดึงโจทย์จาก D5 ให้ Gemini คิดเกณฑ์ บันทึกกลับลง D5

**▶ Process 4.0 ดำเนินการสอบ**
- 4.1 ดึงข้อมูลข้อสอบ: ดึงข้อสอบ(D4) โจทย์(D5) ส่งให้ผู้เรียน (อิง D1)
- 4.2 บันทึกร่างคำตอบ: ผู้เรียนพิมพ์คำตอบ บันทึกลง D7
- 4.3 จัดเก็บรูปภาพคำตอบ: ผู้เรียนอัปโหลดภาพคำตอบให้ Cloudinary บันทึกลิงก์ลง D7
- 4.4 บันทึกสถานะการส่ง: ผู้เรียนกดยืนยัน อัปเดตสถานะสุดท้ายใน D6 และ D7

**▶ Process 5.0 ประเมินผลคำตอบด้วย AI**
- 5.1 ตรวจสอบความพร้อมการประเมิน: อ่านข้อมูลเพื่อ Validate ความพร้อมจากข้อสอบ(D4) โจทย์(D5) สถานะ(D6) คำตอบ(D7)
- 5.2 ส่งคำขอตรวจประเมิน AI: อ่านโจทย์(D5) คำตอบ(D7) จัดเตรียมและส่งคำขอไป Gemini API
- 5.3 ประมวลผลผลการประเมิน: รับผลลัพธ์จาก Gemini อ่าน D7
- 5.4 บันทึกผลการประเมิน: อัปเดตคะแนนรวมลง D6 และรายข้อ/คอมเมนต์ลง D7

**▶ Process 6.0 จัดการผลคะแนนและรายงาน**
- 6.1 ตรวจสอบผลการประเมิน: อ่านผลจาก D6, D7 ส่งให้ผู้สอน
- 6.2 บันทึกการแก้ไขคะแนน: รับคะแนนแก้ไขจากผู้สอน บันทึกลง D7
- 6.3 อนุมัติผลคะแนน: รับสถานะอนุมัติจากผู้สอน บันทึกลง D6, D7
- 6.4 เผยแพร่ผลคะแนน: ดึงข้อมูลที่อนุมัติจาก D6, D7 ส่งให้ผู้เรียนดู

**▶ Process 7.0 จัดการการสื่อสารและการแจ้งเตือน**
- 7.1 สร้างรายงานสรุปคะแนน: ผู้สอนระบุเงื่อนไข ระบบดึง D4, D6, D7 ออกเป็นรายงาน
- 7.2 สร้างรายงานส่วนบุคคล: ผู้เรียนระบุเงื่อนไข ระบบดึง D4, D6, D7 แสดงผลส่วนตัว
- 7.3 แจกจ่ายการแจ้งเตือน: บันทึกแจ้งเตือนลง D12 และส่งให้ผู้ใช้
- 7.4 ปรับปรุงสถานะการอ่าน: อัปเดตการอ่านลง D11, D12
