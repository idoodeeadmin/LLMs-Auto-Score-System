# Project Roadmap & Implementation Plan: Evaly AutoScore System

รายการสิ่งที่ควรทำเพื่อพัฒนาโปรเจกต์ให้สมบูรณ์และพร้อมใช้งานจริง (Production-ready) โดยแบ่งตามความสำคัญและกลุ่มผู้ใช้งาน

## Phase 1: Teacher Analytics & Reporting (ความสำคัญ: สูงมาก)

เพื่อให้ครูสามารถสรุปผลและนำข้อมูลไปใช้งานต่อได้ง่ายขึ้น

### 1) Dashboard สรุปผลรายห้องเรียน
- แสดงค่าเฉลี่ยคะแนน (Mean), มัธยฐาน (Median) และการกระจายตัวของคะแนน
- แสดงจำนวนนักเรียนที่ส่งแล้ว / ยังไม่ส่ง

### 2) ระบบ Export ข้อมูล
- ปุ่ม Export ผลคะแนนนักเรียนทุกคนใน Exam นั้นๆ ออกเป็นไฟล์ CSV หรือ Excel

### 3) Question Difficulty Analysis
- วิเคราะห์รายข้อว่าข้อไหนนักเรียนทำคะแนนได้น้อยที่สุด เพื่อหาจุดบกพร่องในการสอน

### 4) Bulk Actions
- ตรวจสอบความถูกต้องของระบบ Bulk Approve (อนุมัติผลตรวจ AI ทั้งหมดในคลิกเดียว)

## Phase 2: Student Experience & Feedback (ความสำคัญ: สูง)

เพื่อให้นักเรียนได้รับผลประโยชน์สูงสุดจากการตรวจด้วย AI

### 1) หน้า Student Result View
- สร้างหน้า UI ให้นักเรียนดูคะแนนตัวเองหลังจากครู Approve แล้ว
- แสดง Feedback จาก AI และ Comment จากครูแยกตามรายข้อ

### 2) Exam History
- หน้า Dashboard ของนักเรียนที่รวมประวัติการสอบทั้งหมดและคะแนนที่ได้รับ

### 3) Real-time Submission Status
- แจ้งให้นักเรียนทราบว่าข้อสอบกำลังอยู่ระหว่างการตรวจโดย AI (Processing...)

## Phase 3: AI Robustness & Multimodal (ความสำคัญ: กลาง)

เพื่อเพิ่มความแม่นยำในการตรวจข้อเขียนที่ซับซ้อน

### 1) Multi-Image AI Evaluation
- ปรับแก้ Backend ให้ส่งรูปภาพทั้งหมดที่นักเรียนอัปโหลดในข้อเดียวไปให้ Gemini พร้อมกัน (แทนที่จะส่งแค่รูปแรก)

### 2) Prompt Engineering Optimization
- ปรับปรุง System Prompt ให้รอบคอบขึ้น (เช่น ใช้ Chain of Thought เพื่อให้ AI อธิบายเหตุผลก่อนให้คะแนน)

### 3) Token Limit Handling
- เพิ่มระบบจัดการกรณีคำตอบยาวเกินไป หรือรูปภาพมีความละเอียดสูงเกินขีดจำกัดของ API

## Phase 4: Operational Polishing (ความสำคัญ: กลาง)

ความสวยงามและความเป็นมืออาชีพของระบบ

### 1) Exam Timer & Strict Deadlines
- เพิ่มนาฬิกานับถอยหลังในขณะที่นักเรียนทำข้อสอบ
- จัดการ Backend ให้ปฏิเสธการส่งข้อสอบเมื่อเลยเวลา end_date อย่างเข้มงวด

### 2) Empty States & Loading States
- เพิ่มความสวยงามในจุดที่ยังไม่มีข้อมูล (เช่น "ยังไม่มีนักเรียนส่งข้อสอบ")
