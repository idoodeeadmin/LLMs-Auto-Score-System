1.3 ขอบเขตของโครงงาน
ผู้จัดทำได้อธิบายขอบเขตตามประเภทผู้ใช้ 5 ประเภท ดังนี้
1.3.1	ผู้เรียน/นักเรียน
1.3.1.1	 [x] สามารถสมัครสมาชิกได้ด้วยการกรอกข้อมูล
-	ชื่อ
-	นามสกุล
-	รหัสนักเรียน (ถ้ามี)
-	Username (Email)
-	Password
-	รูป Profile (ไม่บังคับ)
1.3.1.2	 [x] สามารถเข้าสู่ระบบได้ด้วยการกรอก
-	Username (Email)
-	Password
1.3.1.3	 [x] สามารถแก้ไขข้อมูลโปรไฟล์
-	ชื่อ
-	นามสกุล
-	Password
-	รูป Profile
1.3.1.4	 [x] สามารถเข้าห้องสอบ (Join Class) ได้โดยการกรอกรหัสคลาส (Class Code) ที่ได้รับจากอาจารย์ และสามารถกดออกจากห้อง/ลบห้อง ได้
1.3.1.5	 [x] สามารถทำข้อสอบและส่งคำตอบได้ 2 รูปแบบ ดังนี้
-	พิมพ์คำตอบผ่านแอปพลิเคชัน
-	อัปโหลดรูปภาพกระดาษคำตอบที่เขียนด้วยลายมือ
1.3.1.6	 [x] สามารถทำข้อสอบภายในระยะเวลาที่กำหนด (Timer Countdown) ระบบจะแจ้งเตือนเมื่อใกล้หมดเวลา (กรณีผู้สอนมีการกำหนดเวลา)
1.3.1.7	 [x] สามารถดูสถานะการส่งคำตอบของงานได้ ดังนี้
-	ยังไม่ส่ง
-	ส่งแล้ว (รอตรวจ)
-	ตรวจแล้ว
1.3.1.8	 [x] สามารถดูคะแนนสอบและอ่านข้อเสนอแนะที่ระบบวิเคราะห์ให้ได้หลังจากอาจารย์อนุมัติผล ให้ Ai
1.3.1.9	 [x] สามารถดูประวัติการทำข้อสอบย้อนหลังได้
1.3.2	ผู้สอน / อาจารย์
1.3.2.1	 [x] สามารถสมัครสมาชิกได้ด้วยการกรอกข้อมูล
-	ชื่อ
-	นามสกุล
-	รหัสผู้สอน (ถ้ามี)
-	Username (Email)
-	Password
-	รูป Profile (ไม่บังคับ)
1.3.2.2	 [x] สามารถเข้าสู่ระบบได้ด้วยการกรอกข้อมูล
-	Username (Email)
-	Password
1.3.2.3	 [x] สามารถแก้ไขข้อมูลโปรไฟล์
-	ชื่อ
-	นามสกุล
-	Password
-	รูป Profile
1.3.2.4	 [x] สามารถ เพิ่ม ลบ แก้ไข ห้องได้
-	สามารถเพิ่มห้องโดยการกรอก ชื่อห้อง
1.3.2.5	 [x] สามารถกรอกรหัสหรือชื่อห้องเพื่อค้นหาได้
1.3.2.6	 [x] สามารถดูรหัสห้องที่ระบบสร้างให้ได้ 
1.3.2.7	 [x] สามารถเพิ่ม แก้ไขโจทย์ข้อสอบภายในห้องได้ โดยกรอกข้อมูล ดังนี้
-	ชื่อข้อสอบ
-	คำถาม/โจทย์ (พิมพ์โจทย์ผ่านแอปพลิเคชัน / อัปโหลดรูปภาพกระดาษคำถาม)
-	ธงคำตอบ หรือ คำตอบ
-	วันและเวลาที่เริ่มสอบ - สิ้นสุดการสอบ (ไม่บังคับ)
-	เกณฑ์การให้คะแนน
-	คะแนนเต็ม
1.3.2.8	 [x] สามารภแก้ไขคะแนนสอบที่มาจาก LLMs ได้
1.3.2.9	 [x] สามารถติดตามสถานการณ์ส่งข้อสอบของนักเรียนได้ (ส่งแล้ว / ยังไม่ส่ง)
1.3.2.10 [x] สามารถตรวจสอบความถูกต้องของคะแนนที่ LLMs ประเมินมาให้ และเพิ่มคอมเมนต์ส่วนตัวได้
1.3.2.11 [x] สามารถกดอนุมัติผลคะแนนเพื่อประกาศให้นักเรียนทราบได้
1.3.2.12 [x] สามารถดูรายงานข้อมูลสถิติคะแนนสอบของนักเรียนได้

1.3.3	แอปพลิเคชันตรวจข้อสอบอัตนัยด้วย LLMs
1.3.3.1	 [x] ระบบจะรองรับการตรวจข้อสอบในรายวิชา โครงสร้างข้อมูล (Data Structures) เท่านั้นเพื่อให้ง่ายต่อการกำหนด rubric หรือ rule-based ได้ชัดเจน และเนื่องจากเป็นวิชาที่ข้อมูลมีรูปแบบคำถามที่แน่นอน เช่น การเขียนโค้ด การวิเคราะห์อัลกอริทึม หรือการทำงานของโครงสร้างข้อมูล (เช่น Stack, Queue, Linked List) ซึ่งเหมาะสมกับการนำมาใช้ในการประเมินผลอัตโนมัติ
1.3.3.2	 [x] รูปแบบข้อมูลนำเข้า (Input Data) ที่ระบบรองรับในรูปแบบ Multimodal จำกัดเฉพาะ "ข้อความ (Text)" และ "รูปภาพ (Image)" เท่านั้น (ไม่รองรับไฟล์เสียงหรือวิดีโอ) 
1.3.3.3	 [x] ระบบรองรับความยาวของคำตอบสูงสุดไม่เกิน 300 คำ ต่อข้อสอบ 1 ข้อ
1.3.3.4	 [x] สามารถแปลงไฟล์รูปภาพลายมือเขียนภาษาไทย/อังกฤษ ให้เป็นข้อความดิจิทัล ได้
1.3.3.5	 [x] สามารถวิเคราะห์ความหมายของคำตอบนักเรียนเทียบกับคำตอบของอาจารย์ โดยไม่อิงแค่คำสำคัญ แต่ดูที่บริบทความหมายได้
1.3.3.6	 [x] ระบบสามารถแจ้งเตือนผู้สอนให้เข้ามาตรวจให้คะแนนเองได้ ในกรณีที่ลายมือของผู้เรียนไม่ชัดเจนจนระบบไม่สามารถวิเคราะห์ผลได้อย่างมั่นใจ 
1.3.3.7	 [x] สามารถเก็บข้อมูลสถิติการสอบในแต่ละรอบได้ ดังนี้
-	รหัสนักเรียน
-	คะแนนที่ได้
-	วันเวลาที่ส่งข้อสอบ
-	ข้อเสนอแนะจากระบบ
1.3.3.8	 [x] ระบบสามารถแจ้งเตือนเมื่อการตรวจข้อสอบเสร็จสมบูรณ์ได้
1.3.3.9	 [x] ระบบสามารถสร้างรหัสห้องไม่ให้ซ้ำกันได้
1.3.3.10 [x] ระบบสามารถส่งคะแนนและข้อเสนอแนะกลับไปให้ผู้สอน อนุมัติได้
1.3.4	เครื่องมือและเทคโนโลยีประมวลผล
1.3.4.1	 [x] ใช้ Multimodal LLM API (เช่น GPT-4o หรือ Gemini 1.5 Pro) ที่มีความสามารถด้าน Vision เป็นตัวประมวลผลหลัก ทั้งการอ่านลายมือจากภาพ และการตรวจทานความถูกต้องของเนื้อหา ในขั้นตอนเดียว โดยมีการออกแบบ Prompt Engineering เพื่อควบคุมให้ AI ตรวจตามเกณฑ์ ที่กำหนดอย่างเคร่งครัด
1.3.4.2	 [x] Frontend: พัฒนา Web Application ด้วย React
1.3.4.3	 [x] Backend: พัฒนาด้วยภาษา Python โดยใช้เฟรมเวิร์ก FastAPI เพื่อรองรับการประมวลผลข้อมูลและการเชื่อมต่อกับ AI Model ได้อย่างมีประสิทธิภาพ ใช้ Node.js สำหรับการแจ้งเตือนสถานะและการนับเวลาถอยหลังแบบ Real-time
1.3.4.4	 [x] Prompt Engineering: ใช้เทคนิค Chain-of-Thought และ Few-Shot Prompting เพื่อควบคุมให้ AI ตรวจตามเกณฑ์รูบริคของวิชา Data Structure อย่างเคร่งครัด
1.3.4.5	 [x] ข้อมูลภาพ คำตอบ และ prompt จะถูกส่งผ่าน API ไปยัง AI Server เพื่อทำการประมวลผล
1.3.4.6	 [x] ใช้ฐานข้อมูล เช่น MySql ในการจัดเก็บโจทย์ คำตอบ ผลคะแนนของผู้เรียน และ UserData
