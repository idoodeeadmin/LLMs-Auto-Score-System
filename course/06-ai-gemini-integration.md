# บทที่ 6: การเชื่อมต่อ AI ด้วย Google Gemini Pro 🧠

มาถึงพระเอกของระบบเรากันแล้ว! **Google Gemini Pro** เป็นโมเดลภาษาขนาดใหญ่ (LLM) แบบ Multimodal แปลว่ามันไม่ได้เข้าใจแค่ "ข้อความ" (Text) แต่มันสามารถเข้าใจ "รูปภาพ" (Vision) ได้ด้วย 

ในระบบ Evaly เราใช้ Gemini ทำ 3 หน้าที่หลักๆ คือ:
1. สร้างเกณฑ์การให้คะแนน (Rubrics) อัตโนมัติจากโจทย์ข้อสอบ
2. อ่านลายมือของนักเรียนจากรูปถ่ายกระดาษคำตอบ
3. ตรวจข้อสอบ ให้คะแนน และเขียน Feedback เชิงลึก

## 🔑 การตั้งค่า Google Gemini API

ในไฟล์ `.env` ที่อยู่ในโฟลเดอร์ฝั่ง Backend เราต้องมีตัวแปร `GEMINI_API_KEY=YOUR_KEY_HERE` 
และในโค้ด Python (`server/utils.py` หรือ `server/services/ai.py`) จะมีการเรียกใช้งานแบบนี้:

```python
import google.generativeai as genai
import os

# 1. โหลด API Key
genai.configure(api_key=os.getenv("GEMINI_API_KEY"))

# 2. เลือกรุ่นของโมเดล (Gemini Pro)
model = genai.GenerativeModel('gemini-pro')
```

## 🗣️ Prompt Engineering (การป้อนคำสั่งให้ AI)

การคุยกับ AI ให้ได้ผลลัพธ์ที่ตรงเป๊ะ ไม่ใช่แค่การทักทายปกติ แต่เราต้องสร้าง **Prompt Template** ที่มีการวางโครงสร้างคำสั่งชัดเจน และบังคับให้ AI ตอบกลับมาในรูปแบบโครงสร้าง JSON เพื่อให้ Python เอาไปบันทึกลงฐานข้อมูลต่อได้ง่าย

**ตัวอย่างโครงสร้าง Prompt สำหรับการสร้างธงคำตอบ (Rubric Generation):**
```python
def generate_rubric_prompt(question_text, full_score):
    return f"""
    คุณเป็นครูผู้เชี่ยวชาญ กรุณาสร้างเกณฑ์การให้คะแนน (Rubric) สำหรับข้อสอบแบบเขียนตอบต่อไปนี้:
    โจทย์: "{question_text}"
    คะแนนเต็ม: {full_score}

    กรุณาวิเคราะห์และแบ่งคะแนนเป็นส่วนๆ พร้อมบอกจุดประสงค์
    
    สำคัญมาก! ห้ามตอบเป็นข้อความปกติ ให้ตอบกลับมาเป็นฟอร์แมต JSON เท่านั้น โดยมีโครงสร้างดังนี้:
    {{
        "criteria": [
            {{"name": "ส่วนที่ 1", "score": 5, "description": "นักเรียนต้องอธิบาย..."}}
        ],
        "model_answer": "ตัวอย่างคำตอบที่ดีที่สุดคือ..."
    }}
    """
```
**เทคนิค:** เราบังคับมันให้ตอบเป็น JSON เพียวๆ เพื่อที่โค้ดของเราจะได้ใช้คำสั่ง `json.loads(ai_response)` ในการดึงค่าไปลงฐานข้อมูลต่อได้ทันที ถ้า AI ตอบมีข้อความทักทายปนมาด้วย โค้ดเราจะ Error ทันทีครับ

## 👁️ Multimodal (การส่งรูปภาพลายมือให้ AI อ่าน)

ฟีเจอร์เด็ดของเราคือนักเรียนสามารถถ่ายรูปคำตอบส่งได้ 
โมเดลที่เราใช้จะเปลี่ยนไปเป็น **`gemini-pro-vision`** หรือในเวอร์ชั่นใหม่ๆ ก็ใช้โมเดลรวมได้เลย

```python
import PIL.Image

def grade_handwritten_exam(image_path, rubric):
    # 1. โหลดรูปภาพด้วยไลบรารี Pillow
    img = PIL.Image.open(image_path)
    
    # 2. สร้าง Prompt รวมกับเกณฑ์
    prompt = f"ช่วยอ่านลายมือในภาพนี้ แล้วประเมินให้คะแนนตามเกณฑ์ต่อไปนี้: {rubric}"
    
    # 3. ส่งทั้ง ข้อความ และ รูปภาพ ไปพร้อมกัน!
    model = genai.GenerativeModel('gemini-1.5-pro')
    response = model.generate_content([prompt, img])
    
    return response.text
```

### ☁️ Cloudinary (ที่พักรูปภาพ)
ก่อนหน้านี้ นักเรียนจะอัปโหลดรูปมาที่เซิร์ฟเวอร์ของเรา แต่ถ้าเซิร์ฟเวอร์เราดับ รูปก็หาย! 
โปรเจกต์เราเลยใช้ **Cloudinary** (บริการรับฝากรูปภาพผ่าน Cloud) โดย:
1. React ส่งรูปให้ Backend
2. Backend ส่งรูปไปฝากที่ Cloudinary
3. Cloudinary คืนค่า URL ของรูปภาพกลับมา (เช่น `https://res.cloudinary.com/.../img123.jpg`)
4. เราเอา URL เนี้ย ไปเซฟลงฐานข้อมูล TiDB
5. เวลาจะตรวจข้อสอบ เราก็ดึงรูปจาก URL นั้นมาประมวลผลครับ

ระบบ Backend ของเราตอนนี้ทำงานได้ครบถ้วนแล้ว! แต่เอ๊ะ... ถ้า AI ตรวจข้อสอบใช้เวลา 10 วินาที หน้าเว็บ React ต้องโหลดหมุนค้างรอ 10 วินาทีเลยหรอ? ไปดูทางแก้ปัญหาในบทที่ 7 กันครับ!
