## 📦 Docker คืออะไร?

ลองจินตนาการว่าโปรเจกต์ Evaly เป็น "อาหาร" 
- ถ้าเราเอาอาหารให้เพื่อน เพื่อนอาจจะต้องมีเตาอบ (NodeJS) หรือมีเตาแก๊ส (Python) อุณหภูมิต้องเป๊ะเหมือนครัวเรา ถึงจะทำอาหารออกมารสชาติเดียวกัน
- **Docker** คือ "กล่องอาหารสำเร็จรูป (Container)" ที่แพ็คทั้งโค้ดเรา, เวอร์ชั่น Python, เวอร์ชั่น NodeJS, และการตั้งค่าทุกอย่าง ปิดผนึกไว้!
- เพื่อนแค่หยิบกล่องนี้ไปเข้าไมโครเวฟ (คำสั่ง `docker run`) แล้วก็ได้กินเลย! อาหารจะรสชาติเหมือนกันเป๊ะ 100% ไม่ว่าเครื่องนั้นจะใช้ Windows, Mac หรือ Linux

### ทำความรู้จัก 2 ไฟล์สำคัญ: `Dockerfile` และ `docker-compose.yml`

#### 1. `Dockerfile` (สูตรทำอาหาร)
ไฟล์นี้เป็นเหมือนกระดาษสูตรอาหารที่สั่งให้ระบบสร้างกล่องคอนเทนเนอร์ (Image) ของ Backend
```dockerfile
# 1. เอา Windows/Linux เปล่าๆ ที่ลง Python 3.9 ไว้แล้วมาเป็นฐาน
FROM python:3.9-slim

# 2. สร้างโฟลเดอร์ทำงาน
WORKDIR /app

# 3. ก๊อปปี้ไฟล์ลิสต์แพ็กเกจ แล้วสั่งติดตั้ง
COPY server/requirements.txt .
RUN pip install -r requirements.txt

# 4. ก๊อปปี้โค้ดเซิร์ฟเวอร์ทั้งหมดของเราใส่ลงไป
COPY server/ .

# 5. สั่งให้รัน Uvicorn แบบเดียวกับที่เราพิมพ์เอง
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

#### 2. `docker-compose.yml` (เมนูเซ็ตจัดเต็ม)
ในโปรเจกต์เรามีทั้ง Frontend, Backend, Database และ Socket.io การจะมานั่งสั่งรันกล่อง (Container) ทีละใบมันเหนื่อย เราเลยใช้ **Docker Compose** เพื่อเขียนสคริปต์ให้รันทุกอย่างพร้อมกันในคลิกเดียว

```yaml
version: '3.8'

services:
  # กล่องที่ 1: หน้าบ้าน (React)
  frontend:
    build: 
      context: .
      dockerfile: client/Dockerfile
    ports:
      - "80:80" # ผูกพอร์ต 80 ของคอม เข้ากับ 80 ของกล่อง

  # กล่องที่ 2: หลังบ้าน (FastAPI)
  backend:
    build:
      context: .
      dockerfile: server/Dockerfile
    ports:
      - "8000:8000"
    environment:
      - GEMINI_API_KEY=${GEMINI_API_KEY} # ดึง API KEY ของจริงมาใส่

  # กล่องที่ 3: ฐานข้อมูล
  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root_password
      MYSQL_DATABASE: evaly_db
```

เวลาจะเริ่มระบบทั้งหมด เราแค่เปิด Terminal (หรือ CMD) ขึ้นมา พิมพ์:
```bash
docker compose up -d
```
แค่นี้ทุกระบบก็จะถูกจำลองและทำงานเชื่อมต่อกันทันที! และเมื่อเอาไปรันบนเซิร์ฟเวอร์ของคลาวด์ เช่น DigitalOcean หรือ AWS ก็แค่พิมพ์คำสั่งนี้คำสั่งเดียว โปรเจกต์สเกลระดับ Enterprise ของคุณก็พร้อมใช้งานแล้วครับ!
