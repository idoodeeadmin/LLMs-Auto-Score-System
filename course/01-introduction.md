# บทที่ 1: แนะนำภาพรวมสถาปัตยกรรม Evaly 🚀

ยินดีต้อนรับสู่โปรเจกต์ **Evaly (LLMs Auto Score System)** ครับ! โปรเจกต์นี้เป็นตัวอย่างที่ดีมากของการสร้างแอปพลิเคชันยุคใหม่แบบ Full-stack ที่มีทั้ง AI, ระบบ Real-time และการใช้งานฐานข้อมูล 

ในบทนี้ เราจะมาทำความเข้าใจ "ภาพรวม" ว่าระบบแต่ละส่วนคุยกันอย่างไร ก่อนที่เราจะเจาะลึกลงไปเขียนโค้ดในบทต่อๆ ไปครับ

---

## 🏗 สถาปัตยกรรมระบบ (System Architecture)

ระบบของเราแบ่งออกเป็น 4 ชั้น (Layers) เพื่อให้ง่ายต่อการดูแลและขยายสเกลในอนาคต:

### 1. Client Layer (หน้าบ้าน - สิ่งที่ผู้ใช้มองเห็น)
- **เครื่องมือ:** React 18, TypeScript, TailwindCSS, Vite
- **หน้าที่:** เป็นหน้าต่างรับข้อมูลจากผู้ใช้ ไม่ว่าจะเป็นการพิมพ์คำตอบ หรือการอัปโหลดรูปภาพกระดาษคำตอบ หน้าบ้านนี้จะสื่อสารกับหลังบ้านผ่านการเรียก API (HTTP Requests) และรอรับแจ้งเตือนผ่าน WebSockets

### 2. Application Layer (หลังบ้าน - สมองสั่งการ)
- **เครื่องมือ:** FastAPI (Python), Node.js (Socket.io)
- **หน้าที่:** 
  - **FastAPI:** จัดการ Business Logic ทั้งหมด เช่น ตรวจสอบสิทธิ์ผู้ใช้ (Login), ดึงข้อมูลห้องเรียน, รับข้อสอบเพื่อส่งต่อให้ AI
  - **Node.js:** ทำงานเป็น Real-time Server แยกต่างหากเพื่อแจ้งเตือน (Notification) เมื่อ AI ประมวลผลเสร็จแล้ว

### 3. AI & External Services (บริการภายนอก)
- **เครื่องมือ:** Google Gemini Pro, Cloudinary, Firebase Auth
- **หน้าที่:** 
  - **Gemini:** เป็นหัวใจหลักในการอ่านข้อความ อ่านลายมือ (Vision) และให้คะแนน 
  - **Cloudinary:** รับฝากรูปภาพกระดาษคำตอบ
  - **Firebase:** ช่วยจัดการความปลอดภัยในการ Login ด้วยบัญชี Google

### 4. Data Layer (ฐานข้อมูล)
- **เครื่องมือ:** TiDB (MySQL-Compatible)
- **หน้าที่:** เก็บข้อมูลทุกอย่างอย่างเป็นระบบ (Users, Rooms, Exams, Scores)

## 🧠 ทำไมเราถึงเลือกใช้ Stack นี้?

ถ้าสังเกต คุณจะเห็นว่าเราเลือกใช้เครื่องมือที่ทันสมัยมาก เหตุผลก็คือ:

1. **ทำไมต้อง Vite แทน Create React App (CRA)?**
   - CRA ค่อนข้างเก่าและทำงานช้าลงมากเมื่อโปรเจกต์มีขนาดใหญ่ ในขณะที่ **Vite** (ภาษาฝรั่งเศสแปลว่า "เร็ว") ทำงานเร็วมาก! ทันทีที่คุณกด Save โค้ด หน้าเว็บจะอัปเดตแทบจะทันที (Hot Module Replacement) ทำให้พัฒนาได้ไวขึ้น

2. **ทำไมต้อง TypeScript แทน JavaScript?**
   - JavaScript ปกติจะยอมให้คุณทำเรื่องผิดพลาดได้ง่ายๆ เช่น เอาตัวเลขไปบวกกับข้อความ แต่ **TypeScript** จะบังคับให้คุณกำหนด "Type" ให้กับข้อมูล ช่วยจับ Bug ตั้งแต่ตอนเขียนโค้ด ไม่ใช่ตอนที่แอปพังไปแล้ว!

3. **ทำไมต้อง TailwindCSS แทน CSS ปกติ?**
   - ปกติเราต้องคิดชื่อ Class แล้วสลับไปเขียน CSS อีกไฟล์ แต่ Tailwind ให้เราใช้ "Utility Classes" ใส่ลงไปใน HTML ได้เลย เช่น `class="text-red-500 font-bold p-4"` โค้ดจะสั้นลง และไม่มีไฟล์ CSS ขยะที่ไม่ได้ใช้ตกค้าง

4. **ทำไมต้อง FastAPI (Python) แทน Node.js หรือ PHP?**
   - FastAPI ถูกสร้างมาเพื่อทำงานที่ต้องประมวลผลพร้อมๆ กัน (Asynchronous) ได้ดีมาก นอกจากนี้ Python ยังเป็นภาษาอันดับ 1 ในวงการ AI/Machine Learning การใช้ Python ทำ Backend จึงทำให้เราเชื่อมต่อกับ Gemini AI ได้ลื่นไหลที่สุด

## 🎯 สรุป

ระบบ Evaly เป็นการทำงานร่วมกันระหว่างหน้าบ้าน (React) ที่รวดเร็ว และหลังบ้าน (FastAPI) ที่เชื่อมกับ AI ได้เก่ง ในบทต่อไป เราจะเริ่มเจาะลึกเข้าไปดูฝั่งหน้าบ้านกัน ว่าโครงสร้างไฟล์เป็นอย่างไร และทำไมถึงต้องมีการตั้งค่า Config ไฟล์เหล่านั้น!
